# スイスドロー記録分布推定ツール

MTG等のスイスドロー形式トーナメントで、各勝敗記録（X-0, X-1 …）に最終的に何人残るかを推定・可視化するローカルWebツール。賞金分配やカットライン判断の参考に使う。

## 使い方

`index.html` をブラウザで直接開くだけで動作する。サーバー・インストール不要。

## 入力項目

| 項目 | 説明 |
|------|------|
| 総参加者数 | トーナメントの全参加人数 |
| bye保持者数 | 1bye / 2bye / 3bye それぞれの人数 |
| ラウンド数 | 「規定（自動）」または手動指定 |
| リミテッド決勝ドラフト | 9〜16人のラウンド数・カット区分に影響 |
| カット強調表示 | 自動 / Top8 / Top4 / なし |

## 規定ラウンド数（付録E準拠）

bye保持者を実効人数に換算（1bye=2人分、2bye=4人分、3bye=8人分）した上で以下の表を引く。

| 実効人数 | ラウンド数 | 決勝カット |
|----------|------------|------------|
| ≤4 | SE 2R（スイスなし） | なし |
| 5–8 | SE 3R（スイスなし） | なし |
| 9–16 | 4（決勝ドラフトあり）/ 5（その他） | Top8（ドラフトあり）/ Top4（その他） |
| 17–32 | 5 | Top8 |
| 33–64 | 6 | Top8 |
| 65–128 | 7 | Top8 |
| 129–226 | 8 | Top8 |
| 227–409 | 9 | Top8 |
| 410+ | 10 | Top8 |

## 計算ロジック

**ペアダウン勝者前提**で分布を推定する。各ラウンドを以下の手順で処理する。

1. ラウンド開始時、byeを使い切った保持者をプールに合流させる（b bye保持者はラウンド b+1 から参加）
2. 勝ち数の高いグループから順に処理
3. 上位グループから降りてきたペアダウン要員は現グループの相手に勝つ
4. 残りをグループ内でペアリング（floor(n/2) 試合）
5. 奇数余りの1人は下位グループへペアダウン繰越

> 実際の結果はドロップ・意図的な引き分け等により異なる場合がある。

## 出力

- **算出情報**：実効人数・使用ラウンド数・自動カット判定
- **ラウンド別グリッド**：各ラウンド終了後の勝ち数別人数（三角形レイアウト）
- **最終記録テーブル**：最終ラウンド終了時の記録別人数（カット強調あり）
- **累積人数テーブル**：X勝以上の累積人数

## ファイル構成

```
index.html      エントリポイント
css/
  style.css     スタイル（レスポンシブ対応、560px以下でモバイルレイアウト）
js/
  script.js     計算ロジック・描画・自動テスト
```

## 自動テスト

`script.js` 内の `runTests()` がページロード時に実行される。失敗時は DevTools コンソールに `[TEST FAIL]` が出力される。

検証ケース：
- 226人・byeなし・R2終了 → `2-0:57 / 1-1:112 / 0-2:57`
- 実効226人 → 8ラウンド、実効227人 → 9ラウンド
- 通常120人 + 1bye×5人（実効130人）→ 8ラウンド
